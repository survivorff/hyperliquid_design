# Hyperliquid 平台概览

> 一条为交易而生的 L1 链 —— 不是"在别人链上做 DEX"，是"为 DEX 造一条链"。

---

## 1. 什么是 Hyperliquid

Hyperliquid 是一个**自建 Layer 1 区块链 + 全链上永续合约交易所**。

关键定位：
- **不是 dApp**：不是部署在 Ethereum / Solana / Arbitrum 上的智能合约
- **是一条链**：自己的共识（HyperBFT）、自己的执行环境（HyperCore + HyperEVM）
- **为交易优化**：200,000 TPS、0.2 秒最终性、全链上订单簿
- **CEX 级体验**：延迟、深度、功能和 Binance Futures 对标，但完全链上

一句话：**Hyperliquid 试图证明"链上交易所可以和中心化交易所一样快、一样好用"**。

## 3. 为什么 Hyperliquid 重要

### 3.1 它解决了"链上交易所不可能三角"

传统观点认为链上交易所不可能同时做到：
- ✅ 去中心化（链上结算、无需信任）
- ✅ 高性能（CEX 级延迟和吞吐）
- ✅ 好体验（限价单、止损、跟单、一键操作）

之前的尝试：
- **dYdX V3**（StarkEx）：性能好但中心化（链下撮合）
- **GMX**（Arbitrum）：去中心化但性能差（AMM 模式，无订单簿）
- **Drift**（Solana）：折中但受 Solana 性能限制

Hyperliquid 的答案：**不在通用链上做，自己造一条链**。

### 3.2 零 VC 融资 + 31% 空投

Hyperliquid 没有拿过一分钱 VC 投资。创始人 Jeff Yan（前 Citadel / Hudson River Trading 量化交易员）用自有资金启动。

2024 年 11 月的 Genesis Distribution：
- **31% 的 HYPE Token 空投给早期用户**
- 没有 VC 份额、没有 advisor 份额
- 这是加密史上最大的"纯用户空投"之一

**为什么这很重要**：
- Token 持有者 = 用户，不是投资人
- 没有解锁抛压（VC 通常 1-2 年后解锁大量 Token）
- 社区忠诚度极高（用户觉得"这是我们的平台"）

### 3.3 从"永续 DEX"到"通用金融链"

2024 年：Hyperliquid = 永续合约交易所
2025 年：Hyperliquid = 永续 + 现货 + HyperEVM（智能合约平台）
2026 年：Hyperliquid = **链上金融基础设施**（借贷、期权、结构化产品都在上面）

HyperEVM 的推出让 Hyperliquid 从"一个产品"变成"一个平台"。任何人都可以在上面部署 DeFi 协议，并且**原子性地和 Hyperliquid 的订单簿交互**。

## 4. 产品功能概览

### 4.1 永续合约（核心产品）

- 150+ 交易对
- 最高 40x 杠杆
- 全链上 CLOB（Central Limit Order Book）
- 订单类型：限价、市价、止损、止盈、跟踪止损、TWAP
- 资金费率：每 8 小时结算
- 交叉保证金 + 逐仓保证金

### 4.2 现货交易

- 2024 年底上线
- 和永续共享同一个订单簿引擎
- 支持 HIP-1 / HIP-2 标准的 Token
- 可以用现货持仓作为永续保证金

### 4.3 Vault 系统

- **HLP（Hyperliquidity Provider）**：平台官方做市 Vault
- 用户存入 USDC → Vault 自动做市 → 赚取手续费分成
- 类似 GMX 的 GLP，但策略更复杂（主动做市而非被动 LP）
- 历史年化 ~10-30%（波动大）

### 4.4 HyperEVM

- 完全兼容 EVM 的智能合约层
- 和 HyperCore（订单簿）共享状态
- 可以在智能合约里**原子性地**下单、撤单、查询持仓
- 已有 DeFi 协议部署：借贷、期权、收益聚合器

### 4.5 Hyperps（预发行永续）

- 对尚未上线的 Token 开永续合约
- 类似"预测市场 + 永续合约"的混合体
- 用 Oracle 定价而非现货价格
- 高风险高波动，但提供了独特的投机工具

## 5. 商业模式

### 5.1 收入来源

| 来源 | 说明 |
|------|------|
| **交易手续费** | Maker 0.01%，Taker 0.035%（标准层级） |
| **清算罚金** | 清算时收取的额外费用 |
| **Vault 管理费** | HLP 等 Vault 的绩效分成 |
| **现货上币费** | HIP-1 Token 上线需要拍卖 |

### 5.2 费用对比

| 平台 | Maker | Taker | 说明 |
|------|-------|-------|------|
| **Hyperliquid** | 0.01% | 0.035% | 有 VIP 层级更低 |
| **dYdX V4** | 0.02% | 0.05% | |
| **GMX V2** | 0.05% | 0.07% | 含价格影响 |
| **Binance Futures** | 0.02% | 0.05% | VIP 0 层级 |

Hyperliquid 的费率是链上永续里**最低的**，接近 CEX 水平。

### 5.3 Assistance Fund

Hyperliquid 没有传统的"Treasury"。取而代之的是 **Assistance Fund (AF)**：

- 用平台收入的一部分**在公开市场回购 HYPE Token**
- 回购的 Token **永久销毁**（减少供应）
- 类似股票回购，但链上透明可查
- 截至 2026 年初，AF 已销毁数百万 HYPE

这是 HYPE Token 的核心价值捕获机制：**平台赚得越多 → 回购越多 → 供应越少 → 价格支撑越强**。

## 6. 团队

- **Jeff Yan**（创始人）：前 Citadel Securities / Hudson River Trading 量化交易员，Harvard 数学系
- 核心团队约 **10 人**，极其精简
- 没有公开的 CTO / COO 等头衔
- 团队成员大多来自**高频交易 / 量化背景**

**为什么团队背景重要**：
- 高频交易出身 → 对延迟、订单簿、撮合引擎有极致理解
- 量化背景 → 对做市、风控、清算有一手经验
- 这解释了为什么 Hyperliquid 的产品"感觉像 CEX" —— 因为团队就是从 CEX / 量化公司出来的

| 事件 | 金额 | 说明 |

| VC 融资 | **$0** | 完全自资金，没有任何 VC 轮 |

| HYPE FDV（峰值） | ~$30B | 2024.12 |
| HYPE FDV（2026 Q1） | ~$15-20B | 回调后稳定 |
| 团队持有 | ~23.8% | 锁仓，逐步释放 |

**没有 VC 的意义**：
- 没有"投资人要求上币安"的压力
- 没有"解锁日大量抛售"的风险
- 团队可以完全按自己节奏做产品
- 但也意味着：**没有外部资本背书，全靠产品说话**

## 9. 为什么你应该关注

不管你是交易者、开发者还是研究员：

**如果你是交易者**：
- Hyperliquid 是目前链上永续**流动性最深、费率最低**的选择
- 但要理解 Vault 风险和清算机制（见后续文章）

**如果你是开发者**：
- HyperEVM 是一个新的 DeFi 开发平台
- 和订单簿的原子组合性是独特优势（其他链做不到）
- 但生态还早期，机会多竞争少

**如果你是研究员**：
- Hyperliquid 代表了"应用链"（App-Chain）范式的极致案例
- 它的成功/失败会影响整个行业对"通用链 vs 专用链"的判断
- JELLY 事件暴露的风险值得深入研究
