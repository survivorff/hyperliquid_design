# 竞争格局：Hyperliquid vs dYdX vs GMX vs Vertex vs Jupiter Perp

> 链上永续合约不是只有 Hyperliquid。这篇从架构、性能、经济模型、用户体验四个维度，系统对比 Hyperliquid 和所有主要竞品。

---

## 1. 市场格局总览

### 1.1 链上永续合约市场份额（2026 Q1）

| 平台 | 市场份额 | 月交易量 | 链 | 模式 |
|------|---------|---------|---|------|
| **Hyperliquid** | **60-70%** | $150-200B | 自建 L1 | 链上 CLOB |
| Jupiter Perp | 10-15% | $30-50B | Solana | Oracle + LP |
| dYdX V4 | 8-12% | $20-35B | Cosmos appchain | 链下内存 CLOB |
| GMX V2 | 3-5% | $8-15B | Arbitrum | Oracle + LP |
| Vertex | 2-4% | $5-10B | Arbitrum | 混合 CLOB |
| Drift | 1-3% | $3-8B | Solana | 混合 CLOB + AMM |
| 其他 | 5-10% | $10-20B | 各链 | 各种 |

**关键观察**：
- Hyperliquid 一家独大，份额是第二名的 4-6 倍
- 但从 2025 Q3 峰值 ~80% 回落到 60-70%，竞品在追赶
- Jupiter 增长最快（受益于 Solana 生态爆发）

### 1.2 为什么要关注竞争

Hyperliquid 的领先不是永恒的。历史上：
- dYdX V3 曾经占 50%+ → 被 Hyperliquid 超越
- GMX 曾经是 Arbitrum 上的王者 → 被 V2 迁移和竞品分流
- Serum 曾经是 Solana 上的基础设施 → FTX 倒塌后消亡

**竞争格局的变化通常来自**：
1. 技术代际跃迁（新架构碾压旧架构）
2. 生态迁移（用户跟着流动性走）
3. 黑天鹅事件（安全事故、监管打击）
4. Token 激励战争（用补贴抢用户）

## 2. dYdX V4：Cosmos 应用链路线

### 2.1 架构

| 维度 | dYdX V4 | Hyperliquid |
|------|---------|-------------|
| 链 | Cosmos SDK appchain | 自建 L1（HyperBFT） |
| 共识 | CometBFT（Tendermint 后继） | HyperBFT（Hotstuff 变体） |
| 撮合 | **链下内存 CLOB**（验证者运行） | **链上 CLOB**（原生状态机） |
| 智能合约 | 无（Cosmos 模块） | 有（HyperEVM） |
| 验证者 | ~60 | 16 |
| 最终性 | ~6 秒 | ~0.2 秒 |
| TPS | ~10K | ~200K |

### 2.2 dYdX 的核心差异

**撮合方式不同**：
- dYdX V4 的订单簿在**验证者的内存**中运行
- 订单匹配发生在链下（内存中），只有成交结果上链
- 这是"链下撮合 + 链上结算"的改良版

**Hyperliquid 的撮合是链上原生状态机**：
- 每个订单都是 L1 交易
- 撮合逻辑是共识的一部分
- 更透明，但对链的性能要求更高

**哪个更好？**
- dYdX 的方式更"传统"，验证者负担小
- Hyperliquid 的方式更"纯粹"，完全可验证
- 实际体验上 Hyperliquid 更快（0.2s vs 6s）

### 2.3 dYdX 的优势

1. **更去中心化**：60 个验证者 vs 16 个
2. **Cosmos 生态**：IBC 互操作，可以和其他 Cosmos 链通信
3. **更长的运营历史**：从 V1 到 V4，经历了多次迭代
4. **开源**：代码完全开源，可独立审计

### 2.4 dYdX 的劣势

1. **性能差距明显**：6 秒最终性 vs 0.2 秒
2. **无智能合约层**：不能在上面部署 DeFi 协议
3. **迁移包袱**：V3→V4 迁移丢失了大量用户和流动性
4. **Cosmos 生态孤立**：和 EVM 世界不互通
5. **Token 激励依赖**：大量交易量靠 DYDX 挖矿驱动

### 2.5 dYdX 的未来

- 正在优化性能（目标缩短最终性）
- 探索和 EVM 链的桥接
- 但结构性劣势（无 EVM、无原子组合）短期内无法解决
- **判断**：dYdX 会保持 #2-3 的位置，但很难重回 #1

## 3. GMX V2：Oracle + LP 池模式

### 3.1 架构

| 维度 | GMX V2 | Hyperliquid |
|------|--------|-------------|
| 链 | Arbitrum / Avalanche | 自建 L1 |
| 撮合 | **无**（Oracle 定价） | 链上 CLOB |
| 对手方 | LP 池（GM Pools） | 做市商 |
| 价格来源 | Chainlink Oracle | 内部撮合价 |
| 杠杆 | 100x | 40x |
| 订单类型 | 有限（市价、限价、止损） | 全部 |

### 3.2 GMX 的核心差异

**GMX 没有订单簿**。它的模式完全不同：

```
传统 CLOB（Hyperliquid）：
  买家 ←→ 卖家（通过订单簿撮合）

GMX 模式：
  交易者 ←→ LP 池（Oracle 定价）
  交易者赚的 = LP 亏的
  交易者亏的 = LP 赚的
```

**这意味着**：
- 没有"价格发现"（价格来自 Oracle，不是市场博弈）
- 没有做市商（LP 池就是对手方）
- 没有限价单（Oracle 给什么价就是什么价）
- 大单无滑点（理论上，但有 price impact 费用）

### 3.3 GMX 的优势

1. **简单易懂**：不需要理解订单簿
2. **高杠杆**：100x（Hyperliquid 只有 40x）
3. **LP 收益**：存入资产赚取交易者亏损 + 手续费
4. **无需做市商**：LP 池自动提供流动性
5. **Arbitrum 生态**：和 Arbitrum 上的 DeFi 可组合

### 3.4 GMX 的劣势

1. **无价格发现**：不能"发现"价格，只能"跟随"
2. **Oracle 依赖**：Oracle 延迟 / 操纵 = 系统风险
3. **LP 风险**：交易者集体赚钱时 LP 大亏
4. **OI 上限**：LP 池大小限制了最大未平仓合约
5. **资本效率低**：需要大量 LP 资金才能支撑交易量
6. **无限价单**：专业交易者不来

### 3.5 GMX 的定位

GMX 和 Hyperliquid 不是直接竞争，更像是**不同市场**：

| 维度 | GMX 用户 | Hyperliquid 用户 |
|------|---------|----------------|
| 类型 | 散户、LP 提供者 | 专业交易者、做市商 |
| 诉求 | 简单、高杠杆、LP 收益 | 低费率、深度、订单类型 |
| 交易频率 | 低-中 | 中-高 |
| 单笔金额 | 小-中 | 中-大 |

**判断**：GMX 会继续存在但不会威胁 Hyperliquid 的地位。它们服务不同的用户群。

## 4. Jupiter Perp：Solana 上的增长最快者

### 4.1 架构

| 维度 | Jupiter Perp | Hyperliquid |
|------|-------------|-------------|
| 链 | Solana | 自建 L1 |
| 撮合 | Oracle + LP 池（类 GMX） | 链上 CLOB |
| 价格来源 | Pyth Oracle | 内部撮合价 |
| 杠杆 | 100x | 40x |
| 生态 | Solana DeFi 全栈 | 独立生态 |
| 用户基础 | Solana 原生用户 | 跨链用户 |

### 4.2 Jupiter 的核心优势

1. **Solana 生态红利**：Solana 用户基数大，Jupiter 是默认入口
2. **聚合器效应**：Jupiter 不只做 perp，还做现货聚合、限价单、DCA
3. **用户体验好**：一站式交易（现货 + perp + 桥 + DCA）
4. **增长快**：2025-2026 年月交易量从 $5B 涨到 $30B+
5. **JUP Token 激励**：积极的 Token 激励拉新

### 4.3 Jupiter 的劣势

1. **Oracle 模式天花板**：和 GMX 一样，无价格发现
2. **Solana 性能限制**：虽然快，但仍是通用 VM（和其他应用抢资源）
3. **LP 风险**：同 GMX
4. **不是 CLOB**：专业做市商不来

### 4.4 Solana Alpenglow 的影响

Solana 正在进行共识层大修（Alpenglow），目标：
- Finality 从 ~12 秒压到 ~150ms
- 更公平的交易排序（BAM）

**如果 Alpenglow 成功**：
- Solana 上的 CLOB 协议（Phoenix、Drift）可能逼近 Hyperliquid 的性能
- Jupiter 可能从 Oracle 模式转向 CLOB
- Solana 的用户基数 + 接近 Hyperliquid 的性能 = 真正的威胁

**但**：
- Alpenglow 还没上线（目标 2026 Q3）
- 即使上线，Solana 仍是通用 VM（撮合不是原生状态机）
- 结构性差距仍在，只是缩小了

### 4.5 判断

Jupiter 是 Hyperliquid 最需要警惕的竞争对手：
- 不是因为技术更好（技术上 Hyperliquid 仍领先）
- 而是因为**用户基数大 + 生态完整 + 增长快**
- 如果 Solana Alpenglow 成功 + Jupiter 转 CLOB → 真正的威胁

## 5. Vertex：混合 CLOB 模式

### 5.1 架构

| 维度 | Vertex | Hyperliquid |
|------|--------|-------------|
| 链 | Arbitrum | 自建 L1 |
| 撮合 | 混合（链下排序 + 链上结算） | 链上 CLOB |
| 特色 | 跨链统一账户 | HyperEVM 原子组合 |
| 费率 | 0 Maker / 0.02% Taker | 0.01% / 0.035% |
| 杠杆 | 20x | 40x |

### 5.2 Vertex 的创新

**跨链统一账户**：
- 用户在 Arbitrum、Mantle、Base 等多链上有统一的交易账户
- 不需要桥接资金，直接跨链交易
- 这是 Vertex 的核心差异化

**零 Maker 费**：
- Maker 完全免费（比 Hyperliquid 的 0.01% 还低）
- 吸引做市商

### 5.3 Vertex 的劣势

1. **规模小**：交易量只有 Hyperliquid 的 3-5%
2. **链下排序**：不是完全链上，有中心化成分
3. **Arbitrum 依赖**：受 Arbitrum 性能和排序器限制
4. **生态薄**：没有类似 HyperEVM 的智能合约层

### 5.4 判断

Vertex 是一个有趣的竞品，但规模差距太大。它的跨链统一账户是好创新，但不足以撼动 Hyperliquid 的地位。

## 6. Drift：Solana 上的混合模式

### 6.1 架构

| 维度 | Drift | Hyperliquid |
|------|-------|-------------|
| 链 | Solana | 自建 L1 |
| 撮合 | 混合（CLOB + AMM + JIT） | 链上 CLOB |
| 特色 | JIT（Just-In-Time）流动性 | Gas-free 订单 |
| 杠杆 | 20x | 40x |

### 6.2 Drift 的创新

**JIT（Just-In-Time）流动性**：
- 当用户下市价单时，做市商可以在最后一刻提供流动性
- 类似 Uniswap V3 的 JIT LP，但用在永续合约上
- 结合了 CLOB 和 AMM 的优点

**三层流动性**：
1. CLOB（限价单）
2. AMM（被动流动性）
3. JIT（即时流动性）

### 6.3 Drift 的劣势

1. **Solana 性能限制**：和其他应用抢 Compute Unit
2. **复杂度高**：三层流动性增加了系统复杂度
3. **规模小**：交易量远不如 Hyperliquid
4. **做市商少**：JIT 模式需要专业做市商参与

### 6.4 判断

Drift 的技术创新值得关注（JIT 模式），但在 Solana 通用 VM 上做 CLOB 的结构性限制仍在。如果 Alpenglow 成功，Drift 可能受益最大。

## 7. 潜在威胁：新进入者

### 7.1 Monad 上的 CLOB

**Monad** 是并行 EVM，TPS 目标 10K+。有团队在 Monad 上构建原生 CLOB。

**威胁程度**：中
- 并行 EVM 提升了通用 VM 的天花板
- 但仍然是"在通用 VM 上做 CLOB"
- 结构性劣势（撮合不是原生状态机）仍在

### 7.2 CEX 的链上化

Binance、OKX、Bybit 都在探索"混合模式"：
- CEX 做撮合 + 链上做结算
- 用户可以选择"自托管模式"
- 本质上是"CEX 体验 + 链上结算"

**威胁程度**：中-高
- CEX 有用户基数和流动性优势
- 但"不是真去中心化"是硬伤
- 监管施压时会回退到纯 CEX

### 7.3 Sui / Aptos 上的 DEX

Move 语言链（Sui、Aptos）也有团队做永续 DEX。

**威胁程度**：低
- 生态规模远小于 Solana / EVM
- 没有明显的技术优势
- 用户基数不够

### 7.4 ZK-Rollup 上的 DEX

StarkNet、zkSync 上也有永续 DEX 项目。

**威胁程度**：低-中
- ZK 证明提供更强的安全保证
- 但延迟和吞吐仍不如 Hyperliquid
- 生态还在早期

## 9. Hyperliquid 的护城河

### 9.1 短期护城河（1-2 年）

1. **性能领先**：0.2s 最终性 + 200K TPS，竞品短期追不上
2. **流动性深度**：做市商已经在这里，迁移成本高
3. **用户习惯**：交易者已经适应了 Hyperliquid 的 UI 和 API
4. **Builder Codes 生态**：前端开发者的收入依赖 Hyperliquid

### 9.2 中期护城河（2-5 年）

1. **HyperEVM 生态**：DeFi 协议部署后有迁移成本
2. **HIP-3 市场网络效应**：更多市场 → 更多用户 → 更多市场
3. **Assistance Fund 飞轮**：收入 → 回购 → 价格 → 更多用户
4. **品牌认知**："链上永续 = Hyperliquid"

### 9.3 长期护城河（5+ 年）

1. **金融基础设施地位**：如果成为"链上 CME"，地位很难撼动
2. **跨链结算中心**：其他链的 DeFi 依赖 Hyperliquid 做结算
3. **监管合规先发**：如果率先获得监管认可

### 9.4 护城河的脆弱点

1. **技术可复制**：HyperBFT 的设计思路可以被抄
2. **流动性可迁移**：做市商跟着利润走，不忠诚
3. **用户可切换**：DEX 切换成本比 CEX 低（只需换个网站）
4. **安全事件**：一次重大安全事故可能导致用户大规模流失

## 10. 竞争格局的未来演变

### 10.1 2026 年下半年

**关键变量**：
- Solana Alpenglow 是否按时上线
- Hyperliquid HIP-4 是否成功
- 监管是否出手

**可能的格局**：
- Hyperliquid 维持 55-65% 份额
- Jupiter 增长到 15-20%
- dYdX 稳定在 8-12%
- 新进入者（Monad CLOB）开始出现

### 10.2 2027-2028 年

**两种可能的未来**：

**情景 A：Hyperliquid 巩固领先**
- HyperEVM 生态爆发
- HIP-3/4 带来新品类
- 竞品无法在性能上追平
- 份额稳定在 50-60%

**情景 B：多极化竞争**
- Solana Alpenglow 成功，Jupiter/Drift 追上
- Monad 上出现强力 CLOB
- CEX 链上化分流一部分用户
- Hyperliquid 份额降到 30-40%，但绝对量仍在增长

### 10.3 我的判断

**最可能的结果是情景 A 和 B 之间**：

- Hyperliquid 保持 #1 但份额从 60-70% 降到 45-55%
- Jupiter 成为 #2（受益于 Solana 生态）
- dYdX 保持 #3 但增长停滞
- GMX 逐渐边缘化（Oracle 模式天花板）
- 新进入者分走 10-15% 的长尾

**核心逻辑**：
- "撮合是原生状态机"的架构优势是结构性的，短期内不会被追平
- 但用户基数和生态广度可以弥补技术差距（Jupiter 的路线）
- 最终决定胜负的不是技术，是**谁能成为"链上金融基础设施"的默认选择**

## 11. 对 CEX 的竞争

### 11.1 链上永续 vs CEX 永续

| 维度 | 链上永续（Hyperliquid） | CEX 永续（Binance） |
|------|----------------------|-------------------|
| 日交易量 | $5-15B | $50-80B |
| 渗透率 | ~10-15% | ~85-90% |
| 增长率 | 50-100% YoY | 10-20% YoY |
| 用户类型 | 加密原生、量化、AI agent | 全部 |
| 优势 | 自托管、透明、可组合 | 速度、深度、法币、合规 |

### 11.2 链上永续的渗透率预测

```
2023: ~3-5%
2024: ~5-8%
2025: ~8-12%
2026: ~10-15%
2027: ~15-25%（如果 Alpenglow 成功 + 监管明确）
2030: ~30-50%（长期均衡）
```

**驱动因素**：
- FTX 事件后的信任危机 → 自托管需求
- AI agent 交易 → 需要链上可编程
- 监管压力 → CEX 合规成本上升
- 技术进步 → 链上体验逼近 CEX

**阻碍因素**：
- 法币入金通道 → CEX 仍是主要入口
- 用户教育 → 大多数人不会用钱包
- 监管不确定 → 可能打击链上永续
- 安全事件 → 桥接风险、合约漏洞

### 11.3 长期均衡

我的判断：**链上永续不会完全替代 CEX，但会占到 30-50% 的市场**。

类比传统金融：
- 电子交易没有消灭场内交易，但占了 90%+
- 链上交易不会消灭 CEX，但会占到显著份额
- 两者会长期共存，服务不同用户群

## 12. 总结

### 12.1 竞争格局的核心结论

1. **Hyperliquid 当前领先是结构性的**：自建 L1 + 原生 CLOB 的架构优势短期内无法被复制
2. **但领先不是永恒的**：Solana Alpenglow、Monad、CEX 链上化都是潜在威胁
3. **最大威胁不是技术，是生态**：Jupiter 的用户基数 + 生态广度可能弥补技术差距
4. **GMX 模式有天花板**：Oracle + LP 池不适合大规模专业交易
5. **dYdX 在去中心化维度领先**：但性能和生态劣势明显
6. **CEX 仍是最大的"竞品"**：链上永续的真正对手是 Binance，不是其他 DEX

### 12.2 Hyperliquid 需要做什么

1. **保持性能领先**：继续优化延迟和吞吐
2. **扩大 HyperEVM 生态**：吸引更多开发者和协议
3. **增加验证者**：解决去中心化争议
4. **开源**：增加社区信任
5. **跨链互操作**：不能只靠 Arbitrum 桥
6. **机构接入**：合规、托管、大客户服务

### 12.3 投资者/用户的启示

- **短期（1 年）**：Hyperliquid 仍是链上永续的最优选择
- **中期（2-3 年）**：关注 Solana Alpenglow 和 Jupiter 的演进
- **长期（5 年）**：链上永续市场会多极化，但 Hyperliquid 大概率保持 #1
